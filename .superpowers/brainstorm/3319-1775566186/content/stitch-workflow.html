Plano Stitch × VitrineMOTORS

4 novas skills + 5 melhorias identificadas = pipeline de redesign de alta qualidade

Skills disponíveis
🎨
/design-md NOVO
.stitch/DESIGN.md
Analisa um projeto Stitch e extrai todos os tokens visuais — cores, tipografia, radius, sombras — num arquivo de design system que garante consistência entre todas as telas geradas.
→ Papel aqui: capturar o DNA visual do VitrineMOTORS antes de gerar qualquer tela nova
✨
/enhance-prompt NOVO
antes de generate_screen
Transforma descrições vagas em prompts estruturados com seções numeradas, tokens de cor no formato correto e keywords de UI/UX que o Stitch entende — melhora drasticamente a qualidade do resultado.
→ Papel aqui: obrigatório antes de cada geração — garante que o Stitch produza o que esperamos
🔁
/stitch-loop NOVO
baton file pattern
Construtor autônomo multi-página. Usa um arquivo de "baton" (.stitch/next-prompt.md) para gerar uma página por vez — cada ciclo gera, integra e prepara o próximo prompt automaticamente.
→ Papel aqui: escalar depois de validar a primeira tela — gerar Home, Listings, Detail com consistência
⚡
/stitch-ui NOVO
pipeline completo
Pipeline end-to-end: Stitch gera o design visual → código de produção React/Tailwind é implementado a partir do HTML gerado. Inclui /design-md + /enhance-prompt automaticamente.
→ Papel aqui: implementar cada componente melhorado — não só o HTML do Stitch, mas React de qualidade

🛡️
Estratégia zero-risco
O projeto não tem git inicializado — por isso todo o trabalho com Stitch vai para a pasta .stitch/ dentro do projeto, separada do código. Os designs ficam lá até você aprovar cada um individualmente. Só depois implementamos os componentes React, um por vez, testando após cada mudança.
🗂️ .stitch/ isolado do src/ 👁️ Review visual antes de implementar 🔬 Um componente por vez 🧪 npm run test após cada mudança
